简介

input 维度 （batch_size, sequence, hidden_size) （b, s, h)， 场景主要考虑推理

数据并行

Data Parallelism，DP 从 batch 维度进行切分。把一个 batch 的样本分给不同 GPU，每张卡持有一份完整模型副本。

切分方式比如 round-robin——样本按 idx % N 轮流分给 N 张卡，保证各卡负载均匀。

推理里的 DP 是各副本独立前向、无梯度通信；训练的 DP 才需要 all-reduce 同步梯度。

张量并行

**Tensor Parallelism **，TP 从 hidden维度切分。attention 层一个 GPU 放若干 head，只要在 g 处做 all-reduce 加在一起就行。

先做一个列并行，再做一个行并行。

列并行:把权重A按照列来分割。

行并行:把权重A按照行分割成两部分。

序列并行

Context Parallelism (CP)，推理的序列并行，当单条序列长到一张卡放不下时（比如 128K、1M 上下文），把序列切到多卡协同算 attention。推理主要用在长上下文的 prefill 阶段。下面讲它的实现 Ring Attention。

核心思路

把 Q、K、V 都沿 sequence 维度切成 N 块，每张卡分到一块 Qi 和一块 Ki,Vi 。

问题：attention 是全局的， Qi 要和所有 K、V 算。怎么办？——把 KV 块在卡之间组成一个环（ring）传一圈：

  1. 每张卡用手头的 Qi 和当前 KV 块算一次「局部 attention」，同时把 KV 块发给下一张卡、从上一张卡收一块新 KV（环形流转，通信与计算重叠）。
  2. 收到新 KV 后继续和 Qi 算，用 online softmax（维护 running max/sum）把结果累加上去。
  3. 转满 N−1 步，每张卡的 Qi 就和全部 KV 都算过了，得到完整输出。

通信时序：N-1 步并行轮转

KV 块沿环同步轮转——每步所有卡同时各传各的（不是串行接力）。4 卡为例，Q 不动、KV 整体右移：

  • 因果 mask 下 Q_i 只和 KVj≤i 算，被 mask 的块（如 Q0+KV3）只传不算。由此带来的负载不均可用 zigzag / striped 两种切分方式平衡。

  • 每层 attention 通信 N−1 次（= CP-1）：KV 走遍其余卡需 N−1 跳；各卡并发，总耗时也只 N−1 步。

  • 类型是点对点 send/recv（比 TP 的 all-reduce 轻），且与计算重叠；FFN 逐 token 本地算，不通信。

显存：序列长度近似随卡数线性扩展

每卡序列相关显存 ≈O(s/N) ，所以序列 s 翻倍只需卡数 N 翻倍，每卡显存不变 → 最大序列长度 ∝ 卡数。说「近似」是因权重每卡存完整一份（CP 不切权重）等固定成本不随 N 缩小。

Sequence Parallelism (SP)，沿 sequence 维度切分 TP 没覆盖的 LayerNorm/Dropout/residual ，省的是训练激活值显存，是 TP 的搭档，只用于训练。

资料：https://zhuanlan.zhihu.com/p/4963530231

专家并行

Expert Parallelism (EP)，MoE 专用，把多个 expert 按 expert 维度切到不同 GPU，每张卡只放一部分 expert。

每个 token 经 router（gating）选中 top-k expert，但选中的 expert 往往不在本卡，所以要把 token 路由到 expert 所在的卡：

  1. 第一次 all-to-all：按 router 结果把各 token 发到目标 expert 所在卡（dispatch）。
  2. 各卡用本地 expert 算 FFN。
  3. 第二次 all-to-all：把结果发回 token 原来的卡（combine）。

图中attention是DP，MoE层是EP

为什么是 all-to-all：token 是按序列/batch（s、b 维度）分布的，expert 是按 expert 维度分布的，两套切分维度不同，需要 all-to-all 做维度转换。

https://arxiv.org/abs/2006.16668

部分框架没有原生 all-to-all 时，会用 all-gather + reduce-scatter 组合来实现等价功能。

流水线并行

Pipeline Parallelism (PP)，按 layer（模型层）维度切分，卡间传 activation。

Gpipe。算完前向再算反向。

1F1B、交错流水线 。上面是1F1B，保持进行中的微批次数量里只有4个（与管道深度一样），这样相比Gpipe显存会更小。

layers 16，pp4配置下。1F1B划分：1-4，5-8，9-12，13-16。交错流水线可以1-2，9-10；3-4，11-12；5-6，13-14；7-8，15-16这样划分，可以理解为先把模型划分成几块（2块），再依次划分给GPU。这样需要更多轮1234才能算完（2次）一个前（反）向，但是时间也相应减小了，使得bubble占比更小。
