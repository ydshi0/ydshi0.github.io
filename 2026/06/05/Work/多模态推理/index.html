本文先介绍 Vision Transformer (ViT) 模型的结构，再到视觉-语言（VL）多模态模型，最后介绍 Qwen3-VL、开源推理框架层面的工程优化。

1 ViT 模型

1.1 概况

Vision Transformer (ViT) 由 Google 在论文 “An Image is Worth 16x16 Words” 中提出，核心思想是：

把图像当作"句子"，把图像 patch 当作"token"，直接用标准的Transformer Encoder。

它首次证明了Transformer 架构在大规模图像分类上可以超越 CNN

1.2 与 CNN 的区别

CNN 每层只看局部小窗口，要堆很多层才能"看到"整张图；ViT 第一层就通过自注意力看到全图，更容易建模长距离关系。

CNN计算量随输入增加近似线性，ViT是 O(N²)

2 计算流程

2.1 预处理 (Preprocess)

预处理通常包含：

  1. 解码 (Decode)：图像/视频从压缩格式还原为像素张量（JPEG/PNG → tensor）。

    [3840, 2160] -> [3840, 2160, 3] 多了 RGB 通道维

  2. Resize / Smart Resize：把像素张量缩放到模型可接受的分辨率

  3. Normalize：减均值除方差

2.2 ViT forward

把 H×W×C 的图像切成固定大小的 patch，每个 patch 处理成一个 token

加上 pos embedding 当作 Encoder 输入

原论文做分类任务会插入前置 [CLS] token ， MLP映射到类别 logits。VL模型MLP是升维到LLM隐藏层维度。

Patch Embedding过程

  • 取像素：一个 14×14 的 RGB patch 含 14 × 14 × 3 = 588个像素值。
  • Flatten 拉直：按固定顺序排成一个长度 588 的一维向量。
  • 线性投影：乘一个矩阵 E (588 × D)，压成 D 维向量，即该 patch 的 token。
  • 切分后的 patch 数量（序列长度）为： (H / P) × (W / P) ，其中 P 是 patch 边长

工程上用一个 Conv2d(in=3, out=D, kernel=14, stride=14) 一次性完成"切块 + 拉直 + 投影"

2.3 VL（视觉-语言）多模型

在多模态模型中，ViT 充当 视觉编码器，其输出经过一个 Projector 投影后，作为视觉 token 拼接到 LLM 的输入序列中：

视觉 token 在 LLM 序列中的位置

LLM 的输入，文本与图像按出现顺序混排。输入中的 image 占位符，会被替换成 ViT 输出的视觉 token。

2.4 ViT 推理开销

虽然 ViT 参数量远小于 LLM，但它也是瓶颈：

  1. 高分辨率 → 长序列：3840 × 2160 的图，patch=14 时序列长度约 42000，计算量很大。
  2. 多图 / 视频：单条 prompt 可能含十几张图甚至上百帧视频。
  3. 没有 KV cache 复用：每张新图都要从头算，类似prefill是计算bound

3 Qwen3-VL

3.1 模型组成

Qwen3-VL 由两个子模型组成：

  • ViT：SigLIP-SO-400M，参数量约 400M
  • LLM：Qwen3 系列

3.2 Qwen3-VL 改进

Qwen3-VL 的主要改进：

方向 传统 VL Qwen3-VL
分辨率 固定 (336/448) **原生动态分辨率 **，按图片真实长宽切 patch
位置编码 1D 2D / 3D RoPE (M-RoPE)，支持时间-高-宽三维位置
视频 抽帧后当多图 原生视频建模，时序 patch + 时间维 RoPE
Connector 单层 MLP / Q-Former Patch Merger + MLP，更高效压缩 token
Encoder CLIP ViT-L SigLIP-SO-400M，对比学习 + sigmoid loss，跨语言更强
OCR / Grounding 较弱 显著增强，支持 bbox / 坐标输出

OCR / Grounding：“图里写了什么，分别在哪？”

2D RoPE

1D RoPE：每个 token 有位置 m，对 Q/K 按 (cos mθ, sin mθ) 旋转，attention 内积时自然得到相对位置

2D RoPE：每个 patch 有坐标 (h, w)，把 Q/K 特征维劈成两半——前一半用 h 旋转，后一半用 w 旋转

Connector

  1. 降 token 数：压缩视觉 token ，减轻 LLM prefill 压力。
  2. 对齐维度：把 ViT 的隐藏维（SigLIP 为 1152）投影到 LLM 隐藏维（如 Qwen 7B 的 3584）。

Qwen3-VL 的 Connector = Patch Merger（2×2 空间合并）+ MLP。

合并的是 2D 空间上相邻的 2×2 = 4 个 patch token，保留局部结构。

3.3 deepstack

Qwen3 ViT 传输给LLM的不仅是最后的vision embedding，还有中间hidden states。qwen3 vit一共27层transformer，中间[8, 16, 24]层的输出都会经过 PatchMerger，作为输出。

4 开源框架

VL 推理与纯 LLM 推理最大的不同：多了一个"非自回归"的 ViT 前置阶段。

4.1 通用优化

  1. kernel 优化、混合精度

  2. 预处理异步化：CPU 上的 resize / normalize 放到独立线程池或 worker 进程，避免阻塞 GPU。

  3. 图像缓存：相同图片的视觉 token 做哈希缓存，多轮对话中复用。

  4. 批处理 (batching)：把多张图、多请求的 patch 序列 batch 到一起送入 ViT。

4.2 sglang 的特色：ViT 分离

sglang 在 PD 分离（Prefill / Decode 分离）之上，把 ViT 独立部署在专门的 GPU 节点：

sglang ViT 节点包含 connector

flowchart LR
    A["ViT 节点
(encoder)"] -- "vision embeddings" --> B["Prefill 节点
(prefill)"]
    B -- "KV cache" --> C["Decode 节点
(decode)"]

分离 ViT的好处

  1. 不同最优硬件：ViT 可用算力强但显存小的卡
  2. 独立扩缩容：图密集场景多开 ViT 节点；纯文本场景把 ViT 节点缩容。
  3. 降低 prefill 节点压力：prefill 节点不再被 ViT 占用 SM，TTFT 更稳定。

跨节点传输：使用 RDMA

以 4K 图像 + SigLIP-SO-400M + Qwen 7B 为例：

项 取值 说明
ViT 输出 token 数 (3840/14) × (2160/14) ≈ 274 × 154 ≈ 42,000 connector 之前
合并后 token 数 N 42,000 ÷ 4 ≈ 10,500 经 Patch Merger（2×2）
传输维度 H_llm 3584 MLP 投影到 LLM 隐藏维
字节数 2 FP16

传输数据量 ≈ N’ × H_llm × 2 ≈ 10,500 × 3584 × 2 ≈ 75 MB

  • GPUDirect RDMA直接 GPU-to-GPU，相比走 TCP / gRPC，延迟从毫秒级降到几十微秒级。

4.3 CUDA Graph 难点

CUDA Graph 把一连串 kernel 录制 成固定图，replay 时省掉 CPU launch 开销，前提是 shape 固定。

decode 每次只算一个 token，形状稳定（batch 分桶即可）；ViT 与 prefill 则有两个麻烦：

  • 序列长度 N 不固定：动态分辨率下每张图 patch 数都不同 → 按 shape 分桶，每桶录一张图
  • varlen attention 动态：cu_seqlens 随输入变化，无法录进图 → 分段捕获，attention 之外的规整部分录图，attention 留在图外正常 launch
