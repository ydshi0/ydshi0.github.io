O. 先理清要解决的问题

标准自注意力在推理时有两大开销：

  1. KV Cache 显存：每生成一个 token，都要缓存历史所有 token 的 Key / Value，显存随上下文长度线性增长。
  2. 注意力计算量：score 计算是 O(L2) ，长上下文下急剧变贵。

后面所有变体，本质都是在「减头数 / 减维度 / 减序列长度」这三个方向上做文章。

维度 代表方法 思路
减头数（共享 KV 头） MQA、GQA 多个 Query 头共享一组 KV
减维度（压缩每个 token 的 KV） MLA 把 KV 投影到低秩潜空间再还原
减序列长度（沿序列压缩 / 稀疏） SWA、NSA、DSA、CSA、HCA 只看局部窗口，或把历史压成更少的 KV 条目 / 稀疏选 top-k

一. MHA（Multi-Head Attention）

原始的注意力公式：

Attention(Q,K,V)=softmax(QK⊤dk)V

每个头都有独立的 WQ,WK,WV 投影矩阵。多头则是各头独立计算后拼接再过 WO ：

headi=Attention(Qi, Ki, Vi) MHA(Q,K,V)=Concat(head1,…,headh)WO
  • 优点：表达能力最强，是质量上限的基准。
  • 缺点：KV Cache 最大——每个头都要缓存自己的 K 和 V，长上下文时显存爆炸。

后续所有变体都在「逼近 MHA 的质量，同时砍掉它的 KV 开销」。

二. MQA & GQA

MQA（Multi-Query Attention）

  • 所有 Query 头共享同一组 K 和 V（只有 1 组 KV）。

  • KV Cache 直接降到 MHA 的 1/h 。

GQA（Grouped-Query Attention）

MHA 和 MQA 之间的折中

  • 把 Query 头分成 g 组，每组内共享一组 KV。 g=h → 退化为 MHA； g=1 → 退化为 MQA。

  • KV Cache 降到 MHA 的 g/h 。

三. MLA（Multi-head Latent Attention）

不共享 KV，而是压缩。

  • 把输入投影到一个所有头共享的低秩潜向量 ckv ，缓存时只存这个潜向量。

  • 推理时再用各头独立的权重把潜向量「解压」回各头的 K/V。

效果：KV Cache 大幅下降，质量却能追平甚至略超 MHA。

DeepSeek V4 并没有使用 MLA。

四. SWA（Sliding Window Attention，滑动窗口注意力）

  • 每个 token 只 attend 固定大小的局部窗口（如 4096）。

  • KV Cache 不再随上下文无限增长，封顶在窗口大小。

五. NSA（Native Sparse Attention，原生稀疏注意力）

从预训练阶段就启用稀疏计算。把每个 query 的 KV 拆成三条并行分支，覆盖不同粒度：

  1. Token 压缩（全局）：KV 按块压成粗粒度表示，兜住全局上下文。
  2. Token 选择（细节）：复用压缩分支的注意力分数当重要性打分，挑出最关键的若干原始 KV 块精确计算attention。
  3. 滑动窗口（局部）：保留最近窗口的原始 KV。

三条分支用可学习门控加权融合（而非简单拼接）。

六. DSA（DeepSeek Sparse Attention，DeepSeek 稀疏注意力）

DeepSeek V3.2 引入，2个步骤：

  1. 闪电索引器（Lightning Indexer）：用一个轻量打分器，快速估计当前 query token 与每个历史 token 的相关性。
  2. Top-k 选择 + 稀疏主注意力：只取分数最高的 k 个历史 KV entry 参与主注意力计算，其余直接跳过。

打分器——一个「迷你版注意力打分」：

  • 主注意力的打分是 q⋅k （高维、多头、全精度），很贵。索引器另外学一套专用的小 query / 小 key（维度低、头数少，FP8 低精度），算个轻量点积当相关性分数。

  • 这套小 query/ 小 key 是从 token 的 hidden state 另起一组独立投影直接算出来的，专门为「打分排序」这个任务训练。

  • 索引器只负责「先粗筛出哪 k 个值得算」，只要把相对顺序排对，不必算准

七. CSA & HCA（DeepSeek V4 的长上下文方案）

DeepSeek V4 是逐层交替两种压缩注意力。两者都额外保留一条 128-token 滑动窗口分支，确保最近的 token 用未压缩的精确 KV。

CSA（Compressed Sparse Attention，压缩稀疏注意力）

CSA 是**「先压缩，再选择」两步叠加**：

第一步：压缩

把 KV 沿序列切成固定大小的块（比如4 ）。设每个 token 的 KV向量维度是 d ，一块就是 4 个 d 维向量：

  1. 加块内位置编码：纯压缩会丢掉「谁在前、谁在后」的次序，所以给块内 4 个槽位各配一个可学习的位置嵌入（intra-block positional embedding），按位加到对应 token 的 K/V 上，让压缩器分得清第 1 个和第 4 个 token。
  2. 映射降维：把加好位置编码的 4 个向量拼接成一个 4d 维向量，再过一个线性层 / 小 MLP 投影回 d 维（ 4d→d ）。输出这一条 d 维向量就代表整块。K 和 V 各有自己独立的压缩器。

于是缓存按 4× 缩小：

第二步：选择

在这些压缩条目里，用 DSA 风格的打分器只挑分数最高的 top-k 条参与主注意力，其余跳过：

  • V4-Pro 取 top-1024，V4-Flash 取 top-512。
  • 关键：top-k 是固定上限，不随上下文增长。上下文越长，选中比例越小，注意力计算量被锁死在 k 条。

一句话：4× 是「缓存压多小」（内存，固定倍率）；top-1024 是「真正参与计算的有多少」（计算，固定绝对值），两者正交。

  • 特点：压缩温和 + 稀疏选择，保留更多细节，擅长精确检索长程信息。

HCA（Heavily Compressed Attention，重压缩注意力）

  • 激进压缩：每 128 个 token 压成 1 条 KV 条目（128× 压缩），再在这些「极少的条目」上做标准注意力（不稀疏、全看）。

  • 特点：条目极少，所以即使全看也很便宜。

总结

CSA HCA
压缩率 温和（~4×） 激进（128×）
注意力方式 稀疏（top-k 选择） 标准
擅长 精确检索、保细节 全局覆盖

配套还有 mHC——用来替代残差流，了解即可。
