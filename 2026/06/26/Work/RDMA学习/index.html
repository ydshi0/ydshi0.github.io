〇、前置知识

ibstatus 里 link_layer 字段是判断环境的关键：

  • link_layer: InfiniBand → IB 环境
  • link_layer: Ethernet → RoCE 环境

NIC：网卡

一、RDMA 介绍

RDMA（Remote Direct Memory Access，远程直接内存访问）：让网卡绕过对端的 CPU 和内核，直接读写远程内存。

核心收益是 kernel bypass（绕过内核）+ zero copy（零拷贝）：数据不经过内核协议栈、不在内核与用户态之间反复拷贝，由网卡直接在用户态内存和网络之间搬运，从而做到低延迟、高带宽、低 CPU 占用。

IB 和 RoCE 都是实现 RDMA 的方式：

  • IB 是一套从物理层到传输层完整自成体系的独立网络；
  • RoCE 则是把 IB 的传输层（RDMA 那套机制）搬到以太网上跑。

两者共享同一套编程接口（verbs），但底层链路 / 网络完全不同。

二、IB 与 RoCE 的异同

InfiniBand（IB）

IB 不只是一个协议，而是一整套独立的网络技术栈，跟以太网 / TCP-IP 是平行、互不兼容的两套体系。它自己定义了全部四层：

  • 物理层：专用线缆、连接器、速率（QDR / EDR / HDR / NDR）
  • 链路层：基于 LID 的寻址，credit-based（信用）流控，天生无损
  • 网络层：GID + 子网，靠 Subnet Manager（SM） 集中分配地址和算路由
  • 传输层：可靠传输 + RDMA 语义（QP 队列对）

关键特点：不用 IP、不用 MAC、不用 TCP；链路层信用流控让它开箱即无损、不丢包，这是 RDMA 高效运行的前提。

RoCE（RDMA over Converged Ethernet）

动机：IB 性能好但要单独建网、太贵。RoCE 的做法是保留 IB 的传输层，把下面的 IB 链路层 / 网络层换成以太网 / IP，从而在普通以太网上也能跑 RDMA。

  • RoCEv1：直接封装在以太网帧里（Ethertype 0x8915），只能二层内通信，已基本淘汰。
  • RoCEv2：封装进 UDP/IP（UDP 端口 4791），可路由，能跨三层组网。现在说 RoCE 默认就是指 v2。

两者最关键的工程差异：无损网络谁来保证

  • IB 靠链路层 credit 流控，天生无损，开箱即用。
  • RoCE 跑在以太网上，而以太网默认会丢包，RDMA 对丢包又极其敏感。所以必须把以太网改造成"无损以太网"，依赖两套机制：
    • PFC（Priority Flow Control，优先级流控）：链路级反压，不丢包。
    • ECN + DCQCN（显式拥塞通知 + 拥塞控制算法）：端到端拥塞通知与控制。

共享的部分：verbs / RDMA 语义

无论 IB 还是 RoCE，上层应用看到的都是同一套 verbs API（QP 队列对、CQ 完成队列、MR 内存注册、Send/Recv、RDMA Read/Write）。这意味着应用代码在两种底层上基本可以通用。

硬件差异

  • 网卡：常常是同一张卡。 例如 ConnectX 系列（VPI）可同时支持 IB 和 Ethernet 两种模式。
  • 交换机：完全是两套，不能混用。
    • IB 用 IB 交换机（NVIDIA Quantum，跑 LID 转发 + SM）；
    • RoCE 用支持 PFC/ECN 的以太网交换机。

三、连接的建立：寻址（LID / GID）与 QP

1. 寻址：LID 与 GID

RDMA 用两种地址寻址，建立连接（QP 互联）时绕不开：

  • LID：16 位本地短地址，SM 分配，子网内交换机靠它转发。IB 独有，类比二层 MAC。
  • GID：128 位全局地址，含固化 GUID，可跨子网路由。IB 和 RoCE 都有（RoCE 用它装 IP），类比 IPv6。

2. 连接的基本单位：QP

RDMA 通信的基本单位是 QP（Queue Pair，队列对）。一个 QP 包含两个队列：

  • SQ（Send Queue）：发送队列，放发送类的 WR
  • RQ（Receive Queue）：接收队列，放接收用的 WR

通信双方各自创建一个 QP，然后把两个 QP 配对连接起来，之后数据就在这对 QP 之间流动。

而要把两端的 QP 接起来，双方必须交换参数。

QPN（Queue Pair Number，队列号）

QPN 是本地网卡为每个 QP 分配的唯一编号，24 位。作用类似 TCP 里的端口号——用来在一张网卡上区分 / 寻址成千上万个 QP。一个数据包到达网卡后，网卡靠包头里的目的 QPN 知道"这个包该投递到哪个 QP"。

怎么来的

  • 调用 ibv_create_qp() 创建 QP 时，驱动 / 网卡自动分配一个 QPN，可从 qp->qp_num 读到。
  • 不能自己指定普通 QP 的号（由硬件分配）。

怎么用：必须交换给对端

RDMA 不像 TCP 有内核帮你握手，QP 的连接参数得自己带外（out-of-band）交换——通常通过一条 TCP/socket 连接，或用 rdma_cm 帮你做。要互联，双方至少要互相告诉对方：

  • 自己的 QPN（对端发包时填进"目的 QPN"）
  • 自己的 LID（IB）或 GID（RoCE，本质是 IP/MAC 信息），用于二 / 三层寻址
  • 自己的起始 PSN（可靠传输的核心）

然后各自调用 ibv_modify_qp()，把状态从 INIT → RTR（Ready to Receive）→ RTS（Ready to Send），在 RTR 阶段把对端的 QPN 填进 dest_qp_num。

四、核心原语

RDMA 提供两类语义，区别在于对端 CPU 是否参与。

1. 双边操作（Two-sided）：Send / Recv

  • 类似传统 socket 的收发：发送端 ibv_post_send（Send），接收端必须提前 ibv_post_recv 挂好接收缓冲区。
  • 数据落到对端哪块内存，由接收端决定（它挂的 recv buffer）。
  • 需要对端参与，适合控制消息、握手、小消息、不知道对端地址的场景。

2. 单边操作（One-sided）：RDMA Write / Read

  • 发送端直接指定远程内存地址 + rkey，网卡直接读写对端内存，对端 CPU 完全不感知、不参与，故称"单边"。这是 RDMA 性能优势最突出的地方。
  • 前提：对端要把目标内存通过 ibv_reg_mr 注册，并把 远程地址 + rkey 带外告诉发送端。

3. 关键配套概念

  • MR（Memory Region，内存注册）：用 ibv_reg_mr 把一段内存注册给网卡，得到 lkey（本地用） 和 rkey（给远端用）。注册的本质是把内存 pin 住（防止换页）+ 建立虚拟地址到物理地址的映射，让网卡能直接 DMA。
  • WR / WQE：Work Request，描述一次操作（缓冲区地址、长度、操作类型等），下发到队列里变成 WQE。
  • CQ / CQE：Completion Queue，操作完成后网卡放一个 CQE 进去，应用通过 ibv_poll_cq 轮询（或事件通知）获知完成。

4. 一次单边 RDMA Write 的大致流程

  1. 双方建链，交换 QPN / GID / PSN，QP 进入 RTS。
  2. 对端注册目标 MR，把 远程地址 + rkey 带外发给本端。
  3. 本端 ibv_post_send 一个 Write WR（带本地 lkey、远程地址、rkey）。
  4. 网卡直接把本地内存数据 DMA 写入对端内存，对端 CPU 无感知。
  5. 本端 CQ 出现 CQE，表示写入完成。

五、GPUDirect RDMA

1. GPUDirect RDMA

nvidia-peermem：把 NVIDIA GPU 显存注册到 Linux 的 RDMA 子系统里，让 Mellanox/ConnectX 网卡通过 PCIe peer-to-peer DMA 直接读写 GPU 的 HBM，数据不绕道主机内存。它是实现 GPUDirect RDMA 的关键软件组件，是多机多卡训练 / 推理打满网络带宽的基础。
