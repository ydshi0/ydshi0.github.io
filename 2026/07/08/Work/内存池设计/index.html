面试常考题

整体结构

预分配大块内存，切成固定大小页（如 64KB）。每个页只有两种状态：在 free_list 或在 LRU_list，互斥。

LRU（Least Recently Used）

核心思想： 最近最少使用的数据最先被淘汰。

数据结构： 哈希表 + 双向链表，两者配合实现所有操作 O(1)。

  • 哈希表：O(1) 定位到节点，避免遍历链表
  • 双向链表：O(1) 插入/删除/移动节点，维护访问顺序

三个基本操作：

操作 步骤 复杂度
get(key) 哈希表找到节点 → 移到链表头部 → 返回值 O(1)
put(key, val) 哈希表查是否存在；存在则更新值并移到头部；不存在则插入头部，满了就删尾部 O(1)
淘汰 删除链表尾部节点 → 从哈希表中移除 O(1)

为什么必须双向链表： 删除节点时需要修改前驱的 next 指针，单向链表做不到 O(1) 删除。

核心操作

分配： free_list 摘块 → 写数据 → 插 LRU 头部，O(1)

释放/删除： 从 LRU 摘除 → 归还 free_list，O(1)

访问命中： 移到 LRU 头部（标记为热）

淘汰（free_list 空时）： 从 LRU 尾部摘最冷的块 → 直接复用写入新数据 → 插 LRU 头部

过期清理

惰性删除 + 定期抽样，两者配合：

机制 作用
惰性删除 访问时检查 TTL，过期则删除并返回 cache miss，零额外开销
定期抽样 每轮随机抽一批 key，删除过期的；过期比例 > 25% 则继续抽，直到比例降下来或耗时到上限

惰性保证不读脏数据，抽样保证无人访问的过期 key 不会泄漏内存。不做全量扫描，避免阻塞。

扩展点（追问时展开）

  • 多种大小 → Slab 分级，每种尺寸一个固定块池，请求向上取整到最近的 class
  • 并发 → 每线程本地池（TLS）+ 全局池，减少锁竞争
  • 数据超过一页 → 页表映射，逻辑连续物理离散（类似 vLLM PagedAttention）
  • vLLM 特有优化 → Copy-on-Write（beam search 共享前缀）、Prefix Caching（跨请求复用）、显存不足时 Swap 到 CPU
