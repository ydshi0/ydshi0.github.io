投机推理（Speculative Decoding）

一、投机推理是什么

核心思路

投机解码把生成拆成三步：

  1. Draft（起草）：用一个"便宜"的方式快速生成 y 个 token（draft tokens）。
  2. Verify（验证）：用大模型一次前向（并行）对这 y 个 token 同时打分。
  3. Accept（接受）：通过一个采样校验规则，接受其中前 k 个正确的 token，拒绝后面的并丢弃。

speculative sampling 的拒绝采样规则，可证明整体等价于直接从大模型采样。

发展路径：

  • 让 draft 更准 → 提高接受率 → 提高平均接受长度；
  • 让 draft 更便宜 → 起草本身别拖后腿。

二、draft方法

n-gram

查找重复出现的 n-gram（连续的 n 个词）来进行预测。例如，如果 “the quick brown fox” 经常出现，模型生成 “the quick brown” 之后，它就会预测下一个词是 “fox”。用到的是字典树（Trie）。

独立draft model

用一个独立的小模型当 drafter（比如用 7B 给 70B 起草）

局限：小模型本身也要占用显存和算力，带来额外负担；

Medusa

在大模型上额外添加多个并行解码头，各头独立预测后续不同位置的 token（t+1、t+2、t+3……），不依赖自回归

三、EAGLE-1：在特征层做草稿

核心洞察

EAGLE 不在 token 层面 做草稿，而是在 feature（特征，即 LM head 之前的最后一层 hidden state）层面做自回归预测。

  1. 特征序列比 token 序列更有规律、更容易预测。直接预测下一个 token 不确定性很大，但预测下一个 hidden feature 更平滑、更可学。
  2. 特征级自回归存在不确定性：仅靠当前特征去预测下一个特征是有歧义的——因为下一步实际采样到哪个 token 是随机的。所以必须把"已经采样出的 token"也作为输入，来消除这个歧义。

架构

EAGLE 的 draft 模型极其轻量，本质就是 一层 decoder

  • 输入 = 当前 token 的 embedding ⊕ 上一步的 feature（两者 concat 后过一个 FC 降维）
  • 输出 = 预测的下一个 feature
  • 再把这个 feature 喂给大模型的 LM head（共享）→ 得到 token 分布 → 采样出 token
  • 这个 token 又作为下一步输入，自回归地生成一条 draft 链

四、Tree Attention（树形注意力）

普通投机解码只猜一条链 A → B → C，一旦某处断掉，后面全废。

EAGLE 是猜一棵树（每步保留 top-k 个候选，展开成树状），用 tree attention一次前向并行验证整棵树。

我们希望把树上的节点塞进同一个 batch 并行算。Transformer 默认是因果注意力——每个位置能看到它前面所有位置。但在树里，C 只应该看到祖先 root→A→C，绝不能看到 B、D、E（它们在别的分支）。

解决办法：自定义 attention mask，让每个节点只能 attend 到自己的祖先链。

第 1 步：展平。 把树按某种顺序（如 BFS）拍平成一维序列：

第 2 步：构造 tree mask。 规则：位置 i 能 attend 到位置 j，当且仅当 j 是 i 的祖先或 j == i。

它只保留祖先关系。实现上就是把不可见位置加 -inf（softmax 后变 0）。

第 3 步：位置编码。 每个节点的 position id 是它在树中的深度。因为 C、D 都是第 2 层，逻辑位置相同

五、EAGLE-2：动态草稿树

EAGLE-1 的 draft 树是静态固定的，没考虑上下文难易。EAGLE-2 发现：draft 预测出的置信度（概率）能很好地近似真实接受率，于是用它来动态决定树形状——只改建树算法，不改架构、不重新训练：

  1. Expand：只对累计置信度（根到该节点的概率乘积）最高的节点继续展开，把算力花在最可能被接受的分支上。
  2. Rerank：再按全局置信度排序，只挑 top-m 个节点送进大模型验证（保证仍是合法子树）。

六、EAGLE-3：去掉特征回归，可 scale

EAGLE-1/2 有两个限制：

  1. 特征回归损失成为瓶颈：训练数据增多时效果很快饱和，无法 scale。
  2. 只用最后一层特征，信息不够丰富。

两大改动

  1. 放弃 feature prediction，改为融合多层特征

    EAGLE-3 不再预测"下一个 feature"，从而去掉了特征回归损失。从大模型采集低层、中层、高层三处的 hidden states，拼接（concat）融合作为 draft 的输入。但任然共享LM head

  2. Training-Time Test（训练时模拟多步）

    EAGLE-1/2 训练时是 teacher forcing：每一步的输入都用大模型的真实 feature。但推理时，draft 是把自己上一步的输出当输入，自回归地连猜好几步——训练和推理不一致（exposure bias），误差会累积。

    EAGLE-3 在训练时就模拟多步自回归：让 draft 用自己预测的结果作为后续输入来训练，使训练、推理对齐。

七、MTP（Multi-Token Prediction）

EAGLE 是推理外挂的草稿器，MTP 是训练期就让模型学会多步预测。

标准 LM 的训练目标是"给定前文，预测下一个 token"。MTP 把目标改成"给定前文，同时预测未来 n 个 token"。

在共享的主干（trunk）之上，挂 n 个独立的输出头，并行算出 n 个位置的预测

DeepSeek-V3 MTP

它用的是串行（causal chain）的 MTP 模块——每个模块依次预测下一个 token，并把上一步的表示传给下一步，保留了 token 之间的依赖。

  1. 共享嵌入层（Shared Embedding）： 共享embedding层
  2. 特征融合： 将最后一层的隐状态经过 RMSNorm 后，与采样结果token 的 embedding进行拼接
  3. 线性投影（Linear Projection）： 将拼接后的向量投影到模型的隐藏维度
  4. 独立 Transformer Block： 每个 MTP 模块拥有一个独立的 Transformer block（单层）
  5. 共享输出头（Shared Output Head）： 共享LM head

额外参数： 每个 MTP 的额外参数仅为：1 层 RMSNorm + 1 个线性投影 + 1 层 Transformer block。
